Data Types

In C#, data types are used to specify the type of data that variables can hold. The choice of data type depends on the nature of the data and the operations that need to be performed on it. C# provides several built-in data types, which can be broadly classified into the following categories:

Integers

Integer types represent whole numbers (positive, negative, or zero) without any fractional or decimal part. The size of integer types can vary across different systems. Common integer types in C# include:

Type Dscription Range Suffix
byte 8-bit unsigned integer 0 to 255
sbyte 8-bit signed integer -128 to 127
ushort 16-bit unsigned integer 0 to 65,535
short 16-bit signed integer -32,768 to 32,767
int 32-bit signed integer -2,147,483,648 to 2,147,483,647
uint 32-bit unsigned integer 0 to 4,294,967,295 u
long 64-bit signed integer -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807 l
ulong 64-bit unsigned integer 0 to 18,446,744,073,709,551,615 ul
Floats

Floating-point types are used to represent real numbers with fractional parts. They can represent both positive and negative values. Common floating-point types in C# include:

Type Dscription Range Suffix
float 32-bit single-precision floating point ±1.5e-45 to ±3.4e38 f
double 64-bit double-precision floating point ±5.0e-324 to ±1.7e308 d
decimal 128-bit decimal type ±1.0e-28 to ±7.9228e28 m
Characters and Strings

Character (char) types represent individual characters and are enclosed in single quotes (''). String types are used to store a sequence of characters and are encloed with double quotes ("").

Type Dscription
char 16-bit single unicode character
string a sequence of unicode characters
Void and Null

Void is a keyword that represents the absence of a value. It is used to specify that a method in C# does not return a value. Null is a literal the specifies a null reference, one that does not refer to any object. It is the default value of referance-type variables.
